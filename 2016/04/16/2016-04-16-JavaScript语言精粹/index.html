《JavaScript语言精粹》读书笔记

Posted by Rubicker on 2016-04-16

《JavaScript：The Good Parts》 Douglas Crockford著
正如书名所说，了解一门语言的精华部分，这样才能更好地掌握它，运用它。

精华&语法

空白

JS注释形式：

  • / /包围的块注释
  • 以//开头的行注释
    由于第一种块注释的结束标志*/容易出现在正则表达式中而产生混淆，所以建议采用行注释

标识符

undefined,NaN和Infinity 是可以作为变量名的，也就是可能会被篡改

数字

JS只有一个单一的数字类型，在内部表示为64位的浮点数。不像大多数其他的编程语言，它没有分离出整数类型，即1和1.0是相同的值。

字符串

  • JS在被创建的时候，Unicode是一个16位字符集，所以JS中的所有字符都是16位的。
  • JS没有字符类型，只需创建一个字符串即可。
  • 转义符\允许将正常情况下不被允许的字符插入到字符串中。
  • \u 约定允许指定用数字表示的字符码位 “A” === “\u0041”
  • 字符串有length属性，表示它的长度
  • 字符串是不可改变的，即一旦创建就无法再改变它（重新赋值只能销毁，新建一个）
  • 两个包含着完全相同的字符且字符顺序也相同的字符串被认为是相同的字符串：
    'c' + 'a' + 't' === 'cat' //true

语句

  • 条件语句中被当作假的值： false null undefined 空字符串 数字0和NaN
  • try语句执行一个代码块，并捕获该代码块抛出的任何异常。catch从句定义一个新的变量接收异常对象
  • throw语句抛出一个异常。若它在一个try代码块中，那么控制权会跳到catch从句中；若它在函数中，那么该函数调用被放弃，且控制权会跳到调用该函数的try语句的catch从句中。
    throw语句中的表达式通常是一个对象字面量，包含一个name属性和一个message属性。
  • JS不允许在return关键字和表达式之间换行；
    JS不允许在break关键字和标签之间换行。

表达式


（从上到下优先级递减）

  • typeof产生的值有’number’,’string’,’undefined’,’function’和’object’。其中，若运算对象是一个数组或null，那么结果是’object’，这是错误的！

字面量

  • 对象字面量：一种指定新对象的表示方法。属性名可以是标识符或字符串（被当作字面量名而不是变量名，所以对象的属性名在编译时才能知道？）
  • 数组字面量：一种指定新数组的表示法。

函数

  • 函数字面量定义了函数值
  • 可以指定一个参数列表，这些参数将作为变量由调用时传递的实际参数初始化

对象

对象是属性的容器。属性名可以是包括空字符串在内的任意字符串。属性值可以是除undefined值之外的任何值。

对象字面量

  • 一个对象字面量就是包围在一对花括号中的0或多个“key/value”;
  • 在对象字面量中，如果属性名是保留字，那么需要用引号括住；其余情况则不需要（前提是合法的JS标识符）
  • 属性值也可以是包括另一个对象字面量在内的表达式，即对象是可以嵌套的
  • 名值对只能是 属性名：属性值（带引号）的格式 ；
    各个名值对之间用逗号相隔；
    最后一个名值对后面一定不能加逗号！

检索

  • 检索对象中包含的值

    • 用[ ]后缀（此时属性名必须加引号，但是可以是保留字）
    • 用点表示法（此时属性名必须不能是保留字）

  • 如果不存在，返回undefined

函数

函数对象

对象字面量产生的对象连接到Object.prototype;函数对象连接到Function.prototype(该原型对象本身连接到Object.prototype)

每个函数对象在创建时也随带有一个prototype属性，它的值是一个拥有constructor属性且值为该函数的对象。

函数字面量

函数对象可以通过函数字面量来创建

调用

调用一个函数将暂停当前函数的执行，传递控制权和参数给新函数。

除了声明时定义的形参，调用时每个函数还接收两个附加的参数：this和arguements；this的值取决于调用模式（方法调用模式，函数调用模式，构造器调用模式和apply调用模式）

  • 方法调用模式

    当一个函数被保存为对象的一个属性时，我们称它为一个方法。当一个方法被调用的时候，this被绑定到该对象（very late binding）

  • 函数调用模式

    当一个函数并非一个对象的属性时，那么它被当作一个函数来调用。当函数以此模式调用时，this被绑定到全局对象。（这是设计上的一个错误）

  • 构造器调用模式

    如果在一个函数签名带上new来调用，那么将创建一个隐藏连接到该函数的prototype成员的新对象，同时this将会被绑定在那个新对象

  • Apply调用模式

    apply方法接收两个参数，第一个是将被绑定给this的值，第二个是一个参数数组

参数

调用函数的时候会得到一个隐藏参数，arguements数组（其实不是数组，类数组而已）

这使得编写一个无须指定参数个数的函数成为可能。

返回

return语句可用来使函数提前返回，当return被执行时，函数立即返回而不再执行余下的语句。

一个函数总是会返回一个值，若没有指定返回值，则返回undefined；但是若函数以在前面加上new的方式调用，且返回值不是一个对象，则返回this（即新对象）

异常

给类型添加方法

通过给原型添加方法，使得所有实例共享方法。

添加方法的时候，记得检验这个方法是否已经存在

递归

JS没有对尾递归进行优化（有返回自身调用结果的递归）

作用域

JS中没有块级作用域，只有函数作用域。所以定义在函数中的参数和变量在函数外部都是不可见的。

建议在函数体的顶部声明函数中可能用到的所有变量。

闭包

内部函数可以访问定义它们的外部函数的参数和变量（除了this和arguements）
