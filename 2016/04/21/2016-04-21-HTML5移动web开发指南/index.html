第三章 HTML5规范的本地存储

HTML5本地存储规范中定义了两个API：Web Storage和本地数据库Web SQL Database

Web Storage实际上是HTML4中Cookies存储机制的一个改进版本，其作用是在网站中把有用的信息存储到本地的计算机或移动设备上，然后根据实际需要从本地读取信息

Web Storage提供了两种存储类型API接口：sessionStorage和localStorage;sessionStorage生命周期仅在会话期间有效，而localStorage存储在本地，永久存储，除非用户删除。

移动设备的支持

if(window.localStrorage){
//浏览器支持localStorage
}

if(window.sessionStorage){
//浏览器支持sessionStorage
}

localStorage

  • 安全性方面

    基于域，然和该域内的页面都可以访问localStorage数据；

    存在问题，各个浏览器之间数据是相互独立的，不能跨域访问。

  • API

    • 如何存储一个数据?

      localStorage.setItem("name","John");
//以key/ value 形式存储

    • 如何读取一个数据？

      localStorage.getItem("name");

localStorage.key(1);   //该方法效果同上，但要确定存储列表中只有一个item

    • 如何删除数据？

      localStorage.removeItem("name");  //删除指定key为“name”的键值对item

localStorage.clear();//删除所有键值对

      • 如何存储和读取JSON格式数据？

        var userData = {
      name = "wang wu";
      sex = "male";
      age = "55"
 };

// 存储userData数据
localStorage.setItem("userData",JSON.stringify(userData));
// 读取userData数据并且赋值给新变量
var newUserData = JSON.parse(localStorage.getItem("userData"));
//删除本地item
localStorage.remove("userData");

第四章 移动Web的离线应用

网页缓存与离线应用的区别：

  • 网页缓存依赖于网络存在，而离线应用在离线状态下仍然可以使用
  • 网页缓存主要缓存当前页面相关内容，也仅限于当前页面的读取；离线应用则主要缓存文件，只要设置缓存该文件的页面，都能在离线状态下读取该文件

移动设备的支持

applicationCache和manifest

  • manifest文件

    离线应用包含一个manifest文件，此文件记录那些资源文件需要离线应用缓存，哪些资源需要通过网络访问等信息。

    如何使用？

    manifest文件的MIME类型是text/cache-manifest属性，因此web服务器需要配置MIME类型才能识别manifest文件。

  • applicationCache对象和事件
    这个对象记录本地缓存的各种状态及事件。缓存状态包括6种，可以通过window.applicationCache.status获得。

第五章 移动设备的常见HTML5表单元素

丰富的表单属性

  • form属性

    在HTML4中，form外的<input>不从属于form表单，提交表单时并不会一并提交；
    但是在HTML5中，只需要给外部的<input>增加form属性并指定其值为formID值，外部的<input>就属于form表单范围。既可以脱离form，又能保持从属关系。

  • placeholder属性

    主要作用是当文本框处于未输入状态并且内容为空时给出的提示内容

  • autofocus属性

    指定控件自动获得焦点（一个HTML页面上只能有一个控件具有该属性）

  • List属性和datalist属性

    list属性主要作用是提示文本框输入，提示的数据源由datalist元素提供。

  • autocomplete属性

    自动完成功能 on/off

  • reruired属性

    必填属性。如果某个元素存在required属性且该元素值为空，则无法提交表单

移动web表单的input类型

  • search类型文本

    用于制作搜索框

  • email类型文本

    用于输入邮箱（某些设备点击时可能会切换输入法）

  • number类型文本

    用于输入数字

  • range类型文本

    数值范围输入文本框，提供的是一种滑动输入方式（需要配合min，max，range属性使用）

  • tel类型文本

    用于输入电话号码

  • url类型文本

    用于输入URL地址的文本框类型

媒体查询Media Queries

viewport设置适应移动设备屏幕大小
  • 什么是viewport
    Apple为了解决移动版Safari的屏幕分辨率大小问题，定义了viewport虚拟窗口，主要作用是允许开发者创建一个虚拟的窗口并定义其窗口大小或缩放功能；
    若开发者没有定义这个虚拟窗口
    • Android Browser浏览器默认大小为800px；
    • IE 974px；
    • Opera 850px;
    • Safari 980px

  • 如何使用viewport
    在meta标签的content属性下设置

  • Media Queries语法总结
    @media [media_query] media_type and media_feature
    media_query表示查询关键字，可以使用no和only

    • no表示对后面的样式表达式执行取反操作

    • only关键字作用是让不支持Media Queries的设备但能读取Media Type类型的浏览器忽略这个样式

      对于支持媒体查询的设备来说，若存在only关键字会忽略并根据后面表达式应用样式文件；对于不支持媒体查询的设备会忽略这个样式文件

    media_type作用是指定设备类型，通称为媒体类型。screen即代表显示器，笔记本，移动端等设备
    media_feature作用是定义CSS中的设备特性，大部分移动设备特性都允许接受min/max前缀
    width，height指浏览器窗口高宽；device-width和device-height指移动设备的屏幕分辨率高宽

第七章 Geolocation地理定位

如何使用Geolocation API

Geolocation API是通过window.navigator.geolocation获取对地理定位的访问

首次获取当前位置

通过getCurrentPosition(successCallback,errorCallback,options)获取当前地理位置信息
其中第一个参数是必选参数，其余均为可选，第三个是可选属性参数设置
options选项：

  • enableHighAccuracy
    指定浏览器或移动设备尝试更精确地读取经纬度，默认为false。
  • timeout
    告诉geolocation允许以毫秒为单位的最大时间间隔
  • maximunAge
    当该缓存的位置信息的时间不大于此参数设定值时，应用程序将接受一个缓存的位置信息。单位是毫秒

在调用该方法成功后返回的函数中，可以通过其中的参数对象获取当前用户访问Web页面时的地理位置信息
pos对象包含coords属性和timestamp属性
coords属性：

  • latitude 以十进制数表示的纬度
  • longtitude 以十进制数表示的经度
  • altitude 位置相对于椭圆球面的高度
  • accuracy 以米为单位的经纬度坐标的精度水平
  • altitudeAccuracy 以米为单位的高度坐标精度水平
  • heading 运动方向 通过相对正北做顺时针旋转的角度指定
  • speed 以米/秒为单位的设备当前地面速度
    timestamp属性：
    用于返回创建pos对象时以毫秒为单位创建的时间戳
监视移动设备的位置变化

watchPosition和clearWatch是一对方法
watchPosition语法与getCurrentPosition相同，clearWatch用来清除watchPosition方法的监听

设置data-inline属性值为true，此时button宽度会自适应按钮文本内容和图标组合的长度

具有分组功能的button按钮

默认分组按钮显示是垂直的，可通过设置data-type值使其水平排列
