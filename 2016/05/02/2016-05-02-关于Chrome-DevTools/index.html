关于Chrome DevTools

Posted by Rubicker on 2016-05-02

相当强大的开发者工具，好好利用必然会给前端开发带来极大便利。此文是结合官方文档写下来的。

审查编辑页面及样式

实时编辑DOM节点

审查元素

  • 直接在面板点击
  • 快捷键
    Ctrl + Shift +C开启审查模式
  • 直接在网页中右键某元素，点击inspect或者检查选项

浏览DOM节点

  • 鼠标
    1.点击你想要折叠或者展开的某一行，此时该行高亮（蓝色背景）
    2.若要展开，则双击该行任意位置或者点击该行前方箭头；若要折叠，点击该行前面向下的箭头
  • 键盘
    1.方向键上下键用来选择哪一行
    2.方向键左键折叠该行，右键展开该行

另外，当指向某一行时，该行对应的元素所属的位置会在下方显示

编辑DOM节点及其特性

  • 直接双击某个节点名字或其特性进行编辑
  • 高亮某行，按下Enter，然后按下Tab选择你想要编辑的特性
  • 在想要编辑的那一行右键弹出菜单，选择Add Attribute或者Edit Attribute
  • 在HTML中编辑某个节点及其子节点：
    • 在某行右键单击，在弹出的菜单中选择Edit as HTML 或者 选中某行后按下F2
    • 按下Ctrl +Enter 保存更改
    • 按下Esc 退出编辑器且不保存更改

移动DOM节点

利用鼠标拖拽即可

删除节点

  • 右键某节点，在弹出菜单上选择删除节点
  • 选择某节点，按下Delete键

实时编辑样式

实时编辑CSS样式
在Elements面板中指定要更改样式的元素，然后在侧边栏Style中单击要更改的样式即可。

  • 最顶端的element.style指的是行内样式
  • 1处是CSS文件中直接指定该元素的样式；
  • 2处是用户代理样式表中相关样式；
  • 3处是根据层叠规则被覆盖的样式（用横线划掉）
  • 4处是继承样式inherted from，单击可以跳转到该位置
  • 5处是虽未定义但是在渲染过程中自动计算出来的样式

审查被CSS选择器影响的元素

将鼠标移到某个样式上，此时在视图中会高亮收到影响的元素

增加，禁用或者开启CSS类名

编辑已存在的属性名值对

单击属性名或者值即可进入编辑模式
按Tab切换下一个属性名/值，按住Shift+Tab切换上一个属性名/值

当编辑某个数字属性值时：

  • Up 和 Down使数字值以1为单位增减
  • Alt + Up 和 Alt+Down 使值以0.1为单位增减
  • Shift +Up 和Shift + Down使值以10为单位增减
  • Shift+ Page Up 和Shift + Page Down使值以100为单位增减

增加一个新的属性声明

任意点击空白处即可在该选择器下增加新的属性声明(可在每条属性名值对前勾选是否开启)

如何设置断点

创建manual断点

直接在JS界面点击行号即可创建断点；
可以取消勾选来暂时禁用某个断点；
再次点选行号即可移除断点（右键单击选择Remove breakpoint也可）

XMLHttp请求断点

  • 当Http请求包含特殊字符串时打断

    When the URL of a request contains a specified string. DevTools breaks before the request is sent.

  • 在特定的XMLHttp请求事件之前打断

在没有捕捉的错误对象处打断

When an error occurs within a method, the method creates an object and hands it off to the runtime system. The object, called an exception object, contains information about the error, including its type and the state of the program when the error occurred. Creating an exception object and handing it to the runtime system is called throwing an exception.

Blackbox功能介绍

某些时候需要”拉黑“某些引入的库代码（如jQuery等），Blackbox就是为此而生的。它允许屏蔽指定的JS文件。

  • 如何屏蔽文件？

    • 通过F1进入Settings设置

      有如下几种方式配置
      • 输入文件名称
      • 用正则表达式匹配
        • 包含特定名称的文件，比如/backbone.js$
        • 特定类型的文件，比如.min.js$
      • 输入整个文件夹，比如bower_components
    • 直接在Sources面板中指定

      点击后：

  • 屏蔽后的效果

    • 库代码（被屏蔽的文件）里抛出异常时不会暂停（当设置为Pause on exceptions时）
    • 调试时Stepping into/out/over都会忽略库代码
    • 事件断点也会忽略库代码
    • 库代码里设置的任何断点也不会起作用
      最终的结果就是只会调试应用代码而忽略第三方代码（配置了Blackbox的代码）。
