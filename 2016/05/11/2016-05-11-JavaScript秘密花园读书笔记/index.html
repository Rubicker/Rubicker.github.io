访问属性

两种方式来访问对象属性，点操作符和中括号操作符

中括号操作符在下列两种情况下依然有效：

  • 动态设置属性
  • 属性名不是一个有效的变量名(如属性名中包含空格，或属性名是JS的关键词)

删除属性

删除属性唯一方法是使用delete操作符；
设置属性为undefined或null仅仅移除了属性和值的关联

属性名的语法

对象的属性名可以使用字符串或者普通字符声明

原型

JS是唯一一个被广泛使用的基于原型继承的语言

属性查找

查找某个对象的属性时，JS会向上遍历原型链，直到原型链的顶部(Object.prototype)，若仍未找到返回undefined

原型属性

可以将任何类型的值赋给原型属性(prototype),但是原子类型赋值操作无效。

性能

若一个属性存在于原型链上端，则将使查找时间大大延长；尤其是查找不存在的属性时会遍历整个原型链；

当使用for in循环遍历对象的属性时，原型链上的所有属性都将被访问
注：可以使用Object.hasOwnProperty(propertyName)来过滤

扩展内置类型的原型

绝对不要扩展内置类型的原型，除非是为了和新的JS引擎兼容

hasOwnProperty函数

hasOwnProperty是JS中唯一一个处理属性但是不查找原型链的函数

但是，JS不保护hasOwnProperty被非法占用，那么当一个对象碰巧占用了这个属性，该如何调用呢?

for in 循环

和in操作符相同，for in循环同样在查找对象属性时遍历原型链上的所有属性。

局部变量

JavaScript 中局部变量只可能通过两种方式声明，一个是作为函数参数，另一个是通过 var 关键字声明

名称解析顺序

比如，当访问函数内的 foo 变量时，JavaScript 会按照下面顺序查找：

1.当前作用域内是否有 var foo 的定义。
2.函数形式参数是否有使用 foo 名称的。
3.函数自身是否叫做 foo。
4.回溯到上一级作用域，然后从 #1 重新开始。

核心

为什么不要使用eval

eval 函数会在当前作用域中执行一段 JavaScript 代码字符串。

但是 eval 只在被直接调用并且调用函数就是 eval 本身时，才在当前作用域中执行。

undefined和null

下面的情况会返回 undefined 值：

  • 访问未修改的全局变量 undefined。
  • 由于没有定义 return 表达式的函数隐式返回。
  • return 表达式没有显式的返回任何内容。
  • 访问不存在的属性。
  • 函数参数没有被显式的传递值。
  • 任何被设置为 undefined 值的变量。

为了避免可能对 undefined 值的改变，一个常用的技巧是使用一个传递到匿名包装器的额外参数。 在调用时，这个参数不会获取任何值。

另外一种达到相同目的方法是在函数内使用变量声明。

这里唯一的区别是，在压缩后并且函数内没有其它需要使用 var 声明变量的情况下，这个版本的代码会多出 4 个字节的代码。

setInterval的堆调用

setTimeout 只会执行回调函数一次，不过 setInterval - 正如名字建议的 - 会每隔 X 毫秒执行函数一次。 但是却不鼓励使用这个函数。

隐藏使用 eval
