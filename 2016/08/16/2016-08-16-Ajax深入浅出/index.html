Ajax原理

什么是Ajax

Asynchronous JavaScript and XML(异步JavaScript和XML)

  • 节省用户操作&时间，提高用户体验，减少数据请求
  • 传输获取数据

Ajax特征

  • 异步发送请求
  • 服务器响应的是数据，而不是页面内容
  • 浏览器中的是应用，不是简单视图

创建第一个Ajax程序

  • 创建Ajax对象
    IE7+: XMLHttpRequest 对象均可用
    IE6: ActiveXObject

    也可采用try-catch块来实现XHR对象的创建，此处不赘述。

  • Ajax对象的方法

    • xhr.open()`` 内有三个参数: 打开方式，地址，是否异步(boolean,true表示异步) 打开方式:get将多个数据以“数据名=数据值&”的格式连接，并放在url?后面传入指定地址;post将数据传给请求头发送 URL长度限制，故get不适宜传递较长数据；理论上，post`传递数据无大小限制
    • `xhr.send()``
      按照open中设置的参数发送请求
    • xhr.responseText
      返回类型为字符串
      Ajax请求返回内容存放处

    • 如何在异步情况下获取返回内容
      利用readyState属性(请求状态)及onreadystatechange事件

      • 0(初始化) 还未调用open()方法
      • 1(载入) 已调用send()方法，正在发送请求
      • 2(载入完成) send()方法完成，已收到全部响应内容
      • 3(解析) 正在解析响应内容
      • 4(完成) 解析完成，可以在客户端调用

    • status
      可能返回的不是我们所想要的内容，此时需要利用http服务器状态码判断一下

      • 1开头：消息
      • 2开头：成功
      • 3开头：重定向(包括缓存)
      • 4开头：请求错误
      • 5开头：服务器错误

  • 了解JSON

    • Ajax中x代表xml，那么为何现在都用JSON呢？
      刚开始确实是使用XML作为通信双方的数据交换载体；后来JSON出现，不仅格式简单，且网络传输数据量更小，因此JSON取代了XML的地位。

    • 如何在IE6，7实现JSON

      • 打开JSON官网
      • 进入Github页面，下载json2.js文件并引入项目文件

      • 在json2.js文件中，可以看到如下几段话:

        This file creates a global JSON object containing two methods: stringify and parse.
        This file is provides the ES5 JSON capability to ES3 systems.
        If a project might run on IE8 or earlier, then this file should be included.
        This file does nothing on ES5 systems.

        简要说明几点:

        • 这个JS文件创建了一个包含stringify和parse两个方法的全局JSON对象
          stringify将对象转换为字符串

          parse 将字符串转换为对应对象

        • 针对ES3提供了ES5中JSON的能力

        • 该JS文件不会对ES5产生影响

get方式两个问题

缓存问题

此时若修改php代码echo "你好，你的名字是{$username},年龄为:{$age}"，刷新页面重新获取，发现”你好”没有被更新出来，原因就在与浏览器缓存,由于get方式获取的URL地址没有发生变化，所以浏览器一直会读取缓存文件

解决方法:
在URL地址中加入随机数,如'getInfo.php?username="leo"&age=32&'+new Date().getTime()

中文乱码问题

若传入中文如'getInfo.php?&username=小明'会产生乱码.

解决方法:
使用encodeURI()编码解决 'getInfo.php?&username'+encodeURI('小明')
